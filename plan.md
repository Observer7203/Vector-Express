Резюме проекта Vector Express
Что это: Метапоисковик логистических услуг - платформа для сравнения цен и сроков доставки от разных перевозчиков (DHL, FedEx, UPS, Ponyexpress и др.)
Ключевые функции MVP:
Расчет стоимости - ввод направления, габаритов, веса → получение предложений от 10+ компаний
AI Чат-бот - анонимная коммуникация между заказчиком и исполнителем
Отслеживание - real-time трекинг груза на карте
Список предложений - сортировка по цене, срокам, рейтингу
Конкурентные преимущества (vs Della KZ, ATI.SU, InDriver):
Все виды перевозок (авиа, авто, ЖД, морские) - конкуренты только авто
AI чат-бот - ни у кого нет
Международные перевозчики (DHL, FedEx, UPS)
Постоплата 15 дней вместо предоплаты
Что взять от мировых аналогов:
Freightos: instant booking, сравнение миллионов маршрутов
Flexport: аналитика, real-time tracking, интеграция с FourKites
Shippo: unified multi-carrier API
Технический стек:
Backend: Laravel 11
Frontend: Vue.js 3
AI: OpenAI API
Real-time: WebSockets
План содержит подробную структуру БД, структуру проекта и пошаговый план реализации.

Описание проекта
Vector Express - метапоисковик логистических услуг для эффективной обработки стоимости и сроков доставки. Слоган: "Find your best logistics solution" / "Найди лучшее решение с нами"
Ключевые проблемы отрасли (из презентации):
Трата времени на коммуникацию с несколькими транспортными компаниями для запроса КП
Трата времени на коммуникацию с Заказчиком и подготовку КП (у Исполнителя)
Содержание большого отдела менеджеров по логистике (и у Заказчика, и у Исполнителя)
Коррупция (договоренности между сотрудником Заказчика и самим Исполнителем)
Предлагаемое решение:
Автоматическое формирование КП от нескольких транспортно-логистических компаний с сортировкой по срокам и стоимости
AI Чат-бот для передачи сопроводительных документов, обеспечивающий анонимность между Заказчиком и Исполнителем

План разработки MVP
Фаза 1: Базовая инфраструктура
1.1 Laravel Backend Setup
 Инициализация Laravel 11 проекта
 Настройка базы данных (миграции)
 Конфигурация API аутентификации (Sanctum)
 Настройка CORS для Vue.js frontend
1.2 Vue.js Frontend Setup
 Инициализация Vue 3 + Vite
 Настройка Vue Router
 Настройка Pinia (state management)
 Базовый UI kit (Tailwind CSS или аналог)
Фаза 2: Основные модули
2.1 Модуль пользователей
Таблицы: users, companies, roles
 Регистрация/авторизация (Заказчик/Исполнитель)
 Профиль компании
 Верификация компаний
2.2 Модуль расчета стоимости
Таблицы: shipments, quotes, routes
 Форма создания заявки на перевозку:
Направление (откуда-куда)
Адреса забора и доставки
Габариты и вес груза (с 3D визуализацией)
Категория груза, опасный/хрупкий
 Алгоритм расчета стоимости от разных перевозчиков
 Сортировка по цене/срокам/рейтингу
2.3 Модуль интеграций с перевозчиками
Таблицы: carriers, carrier_rates, carrier_api_configs
 Абстрактный интерфейс для интеграции
 Интеграция DHL API
 Интеграция FedEx API
 Интеграция UPS API
 Интеграция Ponyexpress
 Интеграция с локальными перевозчиками
2.4 Модуль заказов
Таблицы: orders, order_statuses, order_documents
 Создание заказа из предложения
 Отслеживание статуса заказа
 История заказов
Фаза 3: AI Чат-бот
3.1 Чат-система
Таблицы: chats, messages, chat_participants
 Real-time чат (WebSockets)
 Анонимность между Заказчиком и Исполнителем
 Прикрепление документов
3.2 AI Ассистент
 Интеграция с OpenAI API
 Контекст беседы и история
 Автоматические ответы на FAQ
 Помощь в оформлении документов
Фаза 4: Отслеживание
4.1 Трекинг грузов
Таблицы: tracking_events, tracking_locations
 Интеграция с API перевозчиков для получения статуса
 Карта с отображением местоположения
 Push-уведомления о статусе
Фаза 5: Платежи и биллинг
5.1 Финансовый модуль
Таблицы: invoices, payments, commissions
 Генерация счетов
 Комиссия 5% с продаж
 Отложенная оплата (15 дней)
 Интеграция с платежными системами
Структура базы данных (основные сущности)
users
├── id, email, password, role (customer/carrier/admin)
├── company_id, verified_at

companies
├── id, name, inn, address, type (shipper/carrier)
├── rating, verified

shipments (заявки на расчет)
├── id, user_id, origin, destination
├── weight, dimensions, cargo_type
├── is_dangerous, is_fragile

quotes (предложения от перевозчиков)
├── id, shipment_id, carrier_id
├── price, delivery_days, valid_until

orders (оформленные заказы)
├── id, quote_id, user_id
├── status, tracking_number

carriers (перевозчики)
├── id, name, api_type, api_config
├── rating, supported_routes

chats
├── id, order_id
├── participants (user_ids)

messages
├── id, chat_id, user_id (nullable для AI)
├── content, type, attachments
