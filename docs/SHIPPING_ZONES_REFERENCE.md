## Что такое зоны доставки

**Зона доставки** — это географическая область, объединённая по одному или нескольким критериям:
- Расстояние от точки отправления
- Страна или группа стран
- Тарифная категория
- Время доставки

Зоны используются для:
- Расчёта стоимости доставки
- Определения сроков транзита
- Применения специальных надбавок (remote area, fuel surcharge)
- Построения тарифных матриц

### 2. Номерные зоны (Zone 1-9)

**Когда использовать:** Зоны по расстоянию от хаба (как у DHL, FedEx, UPS)

| Код | Описание | Типичное расстояние | Время доставки |
|-----|----------|---------------------|----------------|
| Z1 | Локальная | 0-150 км | 1 день |
| Z2 | Ближняя | 150-300 км | 1-2 дня |
| Z3 | Региональная | 300-600 км | 2-3 дня |
| Z4 | Межрегиональная | 600-1000 км | 3-4 дня |
| Z5 | Дальняя | 1000-1500 км | 4-5 дней |
| Z6 | Удалённая | 1500-2500 км | 5-6 дней |
| Z7 | Континентальная | 2500-5000 км | 6-7 дней |
| Z8 | Межконтинентальная | 5000-10000 км | 7-10 дней |
| Z9 | Глобальная | 10000+ км | 10+ дней |

**Пример для Казахстана (хаб в Алматы):**
```
Z1: Алматы, Алматинская область
Z2: Жамбылская, Туркестанская области
Z3: Карагандинская, Кызылординская области
Z4: Астана, Акмолинская, Восточно-Казахстанская области
Z5: Костанайская, Павлодарская области
Z6: Актюбинская, Западно-Казахстанская области
Z7: Атырауская, Мангистауская области
```

```

┌─────────────────────────────────────────────────────────┐

│ Zone B: Россия, Узбекистан, Кыргызстан                 │
│ Zone C: Китай, Беларусь, Закавказье                    │
│ Zone D: Турция, ОАЭ, Саудовская Аравия                 │
│ Zone E: Германия, Франция, Великобритания, Италия      │
│ Zone F: Япония, Корея, Сингапур, Индия                 │
│ Zone G: США, Канада, Бразилия                          │

└─────────────────────────────────────────────────────────┘
```

### FedEx International Priority

**Система:** Буквенные коды по регионам

```
A = Domestic (внутри страны)
B = Adjacentcountries (соседние)
C = Regional (региональные)
D = European (европейские)
E = Intercontinental (межконтинентальные)
F = Remote (удалённые)
```

### UPS Worldwide Express

**Система:** Номерные зоны по дистанции

```
Zone 1: 0-500 км
Zone 2: 500-1000 км
Zone 3: 1000-2500 км
Zone 4: 2500-5000 км
Zone 5: 5000-10000 км
Zone 6: 10000+ км
```

## Как выбрать систему кодирования

### Рекомендации по типу бизнеса

| Тип перевозчика | Рекомендуемая система | Причина |
|-----------------|----------------------|---------|
| **Локальный** (внутри страны) | Номерные зоны (Z1-Z9) | Важно расстояние от хаба |
| **Региональный** (СНГ) | ISO коды стран | 1 страна = 1 зона, просто |
| **Международный** | Региональные коды | Группировка по тарифам |
| **Глобальный** | Тарифные зоны (A-I) | Как у DHL/FedEx/UPS |

## Использование в Vector Express

### Как зоны применяются в расчёте

```
1. Пользователь вводит маршрут:
   Откуда: Гуанчжоу, Китай (CN)
   Куда: Алматы, Казахстан (KZ)

2. Система определяет зоны:
   Зона отправления: CN (или Z3 у конкретного перевозчика)
   Зона назначения: KZ (или Z1)

3. Поиск тарифа в матрице:
   rate_cards WHERE origin_zone = CN AND destination_zone = KZ

4. Расчёт стоимости:
   Базовая ставка × Множитель зоны × Оплачиваемый вес + Надбавки
```

### Структура данных

```sql
-- Таблица зон
carrier_zones:
  - id
  - carrier_id
  - zone_code      -- "KZ", "Z1", "EUR-1"
  - zone_name      -- "Казахстан", "Локальная", "Западная Европа"
  - country_code   -- "KZ" (ISO код, если зона = страна)
  - description

-- Тарифная карта
carrier_rate_cards:
  - carrier_id
  - origin_zone_id
  - destination_zone_id
  - transport_type
  - rate
  - transit_days_min
  - transit_days_max
```

### Рекомендации для Vector Express

1. **По умолчанию** — использовать ISO коды стран (KZ, RU, CN)
2. **Для крупных перевозчиков** — загружать их официальные зоны через API
3. **Для локальных** — создавать зоны по регионам внутри страны
4. **Автозаполнение** — при выборе страны подставлять ISO код
