## Пошаговый расчёт

### Шаг 1: Расчёт объёмного веса

```
Объёмный вес = (Д × Ш × В) / DIM-фактор
             = (25 × 20 × 20) / 5000
             = 10,000 / 5000
             = 2.0 кг
```

### Шаг 2: Определение тарифицируемого веса

```
Тарифицируемый вес = MAX(Фактический вес, Объёмный вес)
                   = MAX(4.0, 2.0)
                   = 4.0 кг
```

### Шаг 3: Выбор тарифа

Вес 4.0 кг попадает в диапазон **0-20 кг**

## ОБНАРУЖЕННЫЕ БАГИ В РАСЧЁТЕ

### Проблема 1: Некорректное применение надбавки "Негабаритный груз"

**Ожидание**: Надбавка должна применяться только если груз превышает стандартные размеры (например, любая сторона > 120 см)

**Факт**: Надбавка применяется ко всем грузам RAIL и ROAD, даже к стандартным размерам 25×20×20 см

**Влияние**: +$1.94 (RAIL), +$3.24 (ROAD)

### Проблема 2: Некорректное применение надбавки "Опасный груз"

**Ожидание**: Надбавка должна применяться только если `cargo_type = 'dangerous'`

**Факт**: Надбавка применяется ко всем грузам RAIL и ROAD, хотя груз обычный (`cargo_type = 'general'`)

**Влияние**: +$3.24 (RAIL), +$5.40 (ROAD)

### Проблема 3: Разное поведение для AIR

**Факт**: Для AIR надбавки не применяются вообще, хотя логика должна быть одинаковой
